linux mint下如何安装git以及与git相关的操作

1.安装git

在终端中使用使用命令sudo apt-get install git，在完成安装后进行设置，命令行输入：git config --global user.name "your name" and git config --global user.email "your email"。

2.创建版本库

首先创建一个空目录，进入目录后，使用命令：git init，就把这个目录变成了Git可以管理的仓库。

3.添加和提交文件

新建一个文件，然后命令行使用命令：git add filename，将文件添加入仓库，使用git commit -m "some information"

4.工作区的状态

使用git status查看工作区的状态，如果提示有文件被修改了，可以使用命令git diff来查看修改的内容。

5.版本回退

git log查看历史记录，显示的从最近到最远的提交日志，可以加上–pretty=oneline进行简化输出的信息。我们可以看出前面的commit id，这是我们进行恢复以及版本回退时使用的非常重要的东西。在Git中，使用HEAD表示当前的版本，上一个就是HEAD^以及上上一个就是HEAD^^，在往上就是HEAD~100.

回退到上一个版本使用命令git reset --hard HEAD^,当想恢复新的版本时，当当前窗口尚未关闭的时候，可以看到顺着上面找到最新版本的commit id，然后使用git reset --hard <commit id>,当已经关闭电脑的时候，使用git reflog查看我们每一次的命令，这样同样可以找到我们的commit id。

6.工作区与暂存区

工作区就是我们电脑里能看到的目录，工作区有一个隐藏目录.git，这个是Git的版本库。最重要的一个东西就是暂存区，我们使用git add <filename>时，就是将文件存放到暂存区里，然后使用git commit就是将暂存区的所有内容提交到当前分支上。

7.管理修改

git跟踪的是修改，每次修改，如果不add到暂存区，那么commit也就无法提交到分支上。

8.撤销修改

1.当你改了工作区某一个文件的内容，想直接丢弃工作区的修改时，使用命令git checkout -- file

2.当你不但改乱了工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步，第一步使用命令git reset HEAD file，然后就回到场景1，第二步按照场景1操作。

3.当你已经提交了不合适的修改到版本库时，想要撤销本次提交，参考版本回退一节，前提时没有推送到远程库。

9.删除文件

命令git rm用于删除一个文件，如果一个文件已经被提交到版本库中，那么你永远不用担心误删，使用git checkout -- <filename>进行恢复，但是要小心，你只能恢复文件到最新版，你会丢失最近一次提交后你修改的内容。

使用git rm file是同时删掉了工作区的file,并且将删除的动作提交到了暂存区，git checkout -- file表示从暂存区里吧file拎出来，覆盖到工作区。但是，此时暂存区也已经没有了file，是无法还原覆盖到工作区的，正确的做法是git checkout HEAD -- file,或者git checkout HEAD file。
