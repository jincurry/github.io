git 学习(3)

发表于 2017-09-17

linux mint下如何安装git以及与git相关的操作

1.标签管理

git的标签是版本库的一个快照，但是标签的实质就是一个指向commit的一个指针，我们可以快速的查看到我们的标签。从而就可以从历史版本中取出来。

1.1 创建标签

首先使用命令git branch找到我们需要打标签的分支，然后git checkout branch切换到需要打标签的分支上。

然后使用命令git tag v1.0就可以打上一个新的标签了。使用git tag就可以查看所有的标签。默认标签是打在最新提交的commit的位置上，如果忘记在历史某个时间点打上标签，解决方法就是找到历史提交的commit id，然后打上标签就可以了。使用git log --pretty=oneline --abbrev-commit查看历史提交记录，使用git tag <commit id>就可以在历史时间打上标签了。

我们还可以创建带有说明的标签，用-a指定标签名，用-m来指定说明文字，使用git show <tagname>来查看说明文字。

使用git tag -s <tagname> -m "something"来使用PGP来签名标签。但我们需要安装gpg，我们需要进行配置Key。参考GnuPG帮助文档。

1.2 操作标签

使用git tag -d <tagname>来删除标签。推送标签到远程库时，使用命令git push origin <tagname>，一次性推送所有的尚未推送到远程的标签，使用命令git push origin --tags。

删除远程标签的方法：先从本地删除，使用git tag -d <tagname>，然后进行远程标签的删除，使用命令git push origin:refs/tags/<tagname>。

2.自定义Git

2.1 忽略自定义文件

我们在git的根目录下创建一个特殊的.gitignore文件，然后我们将需要忽略的文件名添加进去，Git就会自动忽略这些文件。

忽略文件的原则是：
忽略操作系统自动生成的文件，比如缩略图等；
忽略编译生成的中间文件、可执行文件等，也就是如果一个文件是通过另一个文件自动生成的，那自动生成的文件就没必要放进版本库，比如 Java 编译产生的.class文件；
忽略你自己的带有敏感信息的配置文件，比如存放口令的配置文件。

2.2 配置别名

使用命令git config --global alias.st staus就表示将命令git status更改为命令git st，以后想要查看状态的时候，使用这一个命令就可以了。我们可以用一个形式来定义这个配置别名：git config --global alias.<othername> command。

2.3 配置文件

每个仓库的git配置文件都放在.git/config文件中，而当前的用户的git配置文件放在用户主目录的一个隐藏文件.gitconfig中。配置别名的时候，我们可以直接修改这个文件，当出现出错的时候，我们可以删掉文件进行重新配置。

3.搭建git服务器

搭建git服务器需要准备一台运行linux的机器。

第一步：安装git sudo apt-get install git；

第二步：创建一个git用户，用来运行git服务 sudo adduser git；

第三步：创建证书登录，收集所有的需要登录的用户的公钥，就是他们的id_rsa.pub文件，把所有的公钥导入到/home/git/.ssh/authorized_keys文件里，一行放置一个。

第四步：初始化Git仓库，先选定一个目录作为git仓库，在该目录下输入命令sudo git init --bare sample.git，就初始化完成了一个裸仓库。

第五步：禁用shell登录，出于安全考虑，第二步创建的 git 用户不允许登录 shell，这可以通过编辑/etc/passwd文件完成。找到类似下面的一行git:x:1001:1001:,,,:/home/git:/bin/bash改为：
git:x:1001:1001:,,,:/home/git:/usr/bin/git-shell
这样，git用户可以正常通过 ssh 使用 git，但无法登录 shell，因为我们为git用户指定的git-shell每次一登录就自动退出。

第六步：克隆远程库，使用命令git clone git@server:/catolog/sample.git在各自的用户下进行运行，这样每个用户就可以在本地修改然后推送到远程库上。这样就完成了一个git服务器的一个搭建。

git 学习(2)

发表于 2017-09-16

linux mint下如何安装git以及与git相关的操作

1.远程仓库

首先注册一个GitHub的账号，就可以免费获得Git远程仓库。

第一步我们需要生成SSH key，使用命令ssh-keygen,然后直接按三次回车即可，会在用户.ssh的目录下生成id_rsa和id_rsa.pub，我们只使用id_rsa.pub这一个公钥。

第二步我们登陆GitHub网站，进入设置页面，选择添加”Add SSH Key”，填写任意的title，在key的文本框里填入id_rsa.pub的内容就可以了。

这样我们就可以开始进行远程仓库的学习了。

2.添加远程库

首先我们在GitHub上新建一个仓库，这是一个非常简单的操作，按照网站上提示进行操作就可以了。我们可以从这个仓库克隆出新的仓库，也可以把一个已有的本地仓库与之关联。然后把本地仓库的内容推送到GitHub的仓库。

使用命令git remote add origin git@github.com:yourname/repository.git进行远程仓库的关联。使用命令git push -u origin master进行把本地库的所有内容推送到远程库上。由于远程库是空的，第一次使用了-u的参数。从现在起，当本地进行了提交，我们可以通过命令git push origin master进行将本地master分支的最新修改推送到GitHub上。

3.从远程库克隆

当我们从零进行开发时，最好的方法是先创建远程库，然后再从远程库进行克隆。

使用命令git clone git@github.com:yourname/repository.git，进行从远程库进行克隆到本地。如果多人协作开发时，那么每个人各自从远程克隆一份就可以了。git支持多种协议，包括https，但通过ssh支持的原生git协议最快。

4.分支管理

4.1 创建和合并分支

Git维持一个主分支叫做master，git用master指向最新的提交，再用 HEAD指向master，就能确定当前分支，以及当前分支的提交点。
创建分支使用命令git check -b newbranch，这是表示创建以及切换到新的分支，也就是相当与下面的两个命令，git branch newbranch和git checkout newbranch，可以使用git branch进行查看当前的分支，当前的分支将会被标记成为*号。在分支上完成了工作，我们切换到主分支上git checkout master，使用命令git merge newbranch进行分支工作成果的合并，当合并完成后，我们就可以放心的删除新建的分支，使用git branch -d newbranch。

4.2 解决冲突

合并过程中，并非一次就能完成的，当合并的时候会出现问题，Git会提示我们出现了冲突了，我们需要手动解决冲突后再提交。使用命令git status进行查看冲突的文件。当我们手动解决冲突以后，可以使用git log --graph --pretty=oneline --abbrev-commit也可以看到分支的合并情况。最后再删除不用的分支即可。

4.3 Bug分支

当接到修复bug的任务时，我们自然会去创建一个分支去修复它，但是我们正在进行开发的工作并没有完成，这时，我们可以使用stash的功能就行将工作现场”储藏”起来，等以后恢复现场后继续工作，使用的命令就是git stash。当修复分支完成工作后，切换到dev版进行开发，却发现工作区是干净的。这时候使用命令git stash list就可以进行查看工作现场。恢复有两种方法：第一个就是使用git stash apply进行恢复，但是恢复后，stash内容并没有删除，需要使用git stash drop来删除；第二种就是使用git stash pop，会在恢复的同时将stash内容删除。

4.4 Feature分支

开发一个新的功能的时候，最后新建一个分支，在这个分支上进行开发，最后合并到主分支上完成新的功能的添加。
但是要丢弃一个没有被合并过的分支，可以通过命令git branch -D newbranch来强行删除。

4.5 多人协作

查看远程库的信息使用命令git remote。或者使用git remote -v来显示更加详细的信息。

使用命令git push origin master来进行推送分支，将该分支上的所有的本地提交推送到远程库。如果推送失败，先用git pull抓取远程的新提交。

在本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致。

建立本地分支和远程分支的关联，使用git branch --set-upstream branch-name origin/branch-name。

从远程抓取分支，使用git pull，如果有冲突，要先处理冲突。

git 学习(1)

发表于 2017-09-15

linux mint下如何安装git以及与git相关的操作

1.安装git

在终端中使用命令sudo apt-get install git，在完成安装后进行设置，命令行输入：git config --global user.name "your name" and git config --global user.email "your email"。

2.创建版本库

首先创建一个空目录，进入目录后，使用命令：git init，就把这个目录变成了Git可以管理的仓库。

3.添加和提交文件

新建一个文件，然后命令行使用命令：git add filename，将文件添加入仓库，使用git commit -m "some information"

4.工作区的状态

使用git status查看工作区的状态，如果提示有文件被修改了，可以使用命令git diff来查看修改的内容。

5.版本回退

git log查看历史记录，显示的从最近到最远的提交日志，可以加上–pretty=oneline进行简化输出的信息。我们可以看出前面的commit id，这是我们进行恢复以及版本回退时使用的非常重要的东西。在Git中，使用HEAD表示当前的版本，上一个就是HEAD^以及上上一个就是HEAD^^，在往上就是HEAD~100.
回退到上一个版本使用命令git reset --hard HEAD^,当想恢复新的版本时，当当前窗口尚未关闭的时候，可以看到顺着上面找到最新版本的commit id，然后使用git reset --hard <commit id>,当已经关闭电脑的时候，使用git reflog查看我们每一次的命令，这样同样可以找到我们的commit id。

6.工作区与暂存区

工作区就是我们电脑里能看到的目录，工作区有一个隐藏目录.git，这个是Git的版本库。最重要的一个东西就是暂存区，我们使用git add <filename>时，就是将文件存放到暂存区里，然后使用git commit就是将暂存区的所有内容提交到当前分支上。

7.管理修改

git跟踪的是修改，每次修改，如果不add到暂存区，那么commit也就无法提交到分支上。

8.撤销修改

1.当你改了工作区某一个文件的内容，想直接丢弃工作区的修改时，使用命令git checkout -- file
2.当你不但改乱了工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步，第一步使用命令git reset HEAD file，然后就回到场景1，第二步按照场景1操作。
3.当你已经提交了不合适的修改到版本库时，想要撤销本次提交，参考版本回退一节，前提时没有推送到远程库。

9.删除文件

命令git rm用于删除一个文件，如果一个文件已经被提交到版本库中，那么你永远不用担心误删，使用git checkout -- <filename>进行恢复，但是要小心，你只能恢复文件到最新版，你会丢失最近一次提交后你修改的内容。
使用git rm file是同时删掉了工作区的file,并且将删除的动作提交到了暂存区，git checkout -- file表示从暂存区里吧file拎出来，覆盖到工作区。但是，此时暂存区也已经没有了file，是无法还原覆盖到工作区的，正确的做法是git checkout HEAD -- file,或者git checkout HEAD file。
